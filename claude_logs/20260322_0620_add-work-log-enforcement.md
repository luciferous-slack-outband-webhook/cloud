# 作業ログ書き忘れ防止の仕組み導入

## ユーザーのプロンプト

- CLAUDE.mdに作業履歴を残してと書いたのに書いてくれなかった。確実に作業履歴を残すようにさせるにはどうしたらいいか？
- Hooksでコンテキストを踏まえた作業履歴を残せるのか？
- Skillsで自動化できないのか？
- CLAUDE.mdに書いた内容を確実に履行させるにはどうしたらいいのか？

## 会話履歴（概要）

1. ユーザーがCLAUDE.mdの指示が守られない問題を報告
2. Hooks / CLAUDE.md強化 / Skills の各アプローチを比較検討
3. Hooksはシェルコマンドのみでコンテキスト参照不可、Skillsは手動呼び出しのみであることを説明
4. CLAUDE.mdは「強い指示」であり「強制」ではないこと、確実な履行にはHooksが必要であることを説明
5. ユーザーが「CLAUDE.md強化 + Stop Hook」の方針を選択
6. 計画を作成・承認後、実装を完了

## 作業の理由

CLAUDE.mdに作業履歴の記録指示があるにもかかわらず、Claude Codeが指示を忘れてログを作成しないことがあった。確実にログを残す仕組みが必要。

## 計画内容

CLAUDE.mdの指示強化（CRITICAL表記、先頭配置）とStop Hookによる強制チェックの2層構造で、作業ログの書き忘れを防止する。

## 作業の内容

### 変更ファイル

| ファイル | 操作 |
|---------|------|
| `CLAUDE.md` | 作業履歴セクションを先頭に移動、CRITICAL/WARNING表記で強化 |
| `.claude/settings.local.json` | Stop Hook設定を追加 |
| `scripts/check-claude-log.sh` | 新規作成 - Hook用チェックスクリプト |
| `claude_logs/.gitkeep` | 新規作成 - ディレクトリ確保 |

### Stop Hookの判定ロジック

1. `stop_hook_active=true` なら即スキップ（無限ループ防止）
2. `claude_logs/` 以外のファイル変更がなければスキップ（質問のみの会話）
3. `claude_logs/` に直近10分以内の `.md` があればOK
4. なければ `block` を返してログ作成を促す
